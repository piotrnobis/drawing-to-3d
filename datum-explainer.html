01 / the problem

A drawing is not a model.

Industry runs on millions of legacy 2D drawings with no 3D counterpart. To manufacture a replacement part, someone has to rebuild it by hand in CAD. That is slow, and it is exactly the kind of structured visual reasoning a model should be able to do.

The hard constraint

The output has to be parametric and editable; a STEP B-rep with a real feature tree. A generated mesh looks fine and is useless: you cannot change a wall thickness or re-fit a bore on a triangle soup.

Why it is hard

Inferring one solid that is consistent across three views is genuine spatial reasoning, and off-the-shelf vision models are weak at it. Worse, a part that looks right but measures wrong is worthless. So checking the result is not optional.

02 / how it works

Read the drawing, build the part, prove it is right.

Four phases. The first turns pixels into intent, the last two are where we earn trust: we rebuild the solid back into a drawing and measure it against the original.

03 / the idea that wins

We do not just generate. We check.

Most attempts stop at "produce something that looks like the drawing." Our differentiator is closing the loop with two measurable signals, so the model can see its own mistakes and the judges can see ours are caught.

Signal 1 · shape

Project back, then overlay

We re-draw the solid as an orthographic line drawing and lay it over the original. Mismatches in outline, holes, and steps light up in red; this localises where the part is wrong.

Signal 2 · size

Measure every dimension

We query the actual solid and check each callout against the spec, within tolerance. This is the hard gate, and it is the artifact we put on screen in the demo.

FLAG → refine. The bore is out of tolerance, so it goes back into the loop with that exact number until it passes.

04 / the refine loop

Grounded feedback, not vibes.

The loop hands the model the overlay image and the failing numbers, asks for a corrected part, and re-checks. It is bounded on purpose: refinement plateaus fast, so we cap it and keep the best version we have seen.

max 3 iterations plateau guard · stop if no gain keep best · never regress repair errors first, cheaply
05 / stack & scope
One model

A single vision model (Gemini) drives reading, generation, and refinement. No model zoo to babysit during the hackathon.

Real CAD kernel

CadQuery on OpenCASCADE gives us true B-rep, STEP export, and the orthographic projection we compare against; the same engine validates dimensions.

A UI that demos

A clean web app shows every stage live: drawing in, spec overlay, 3D viewer, diff, and the pass/fail table. The pipeline is the story.

Scope we are honest about. We win by nailing a few real part classes end to end (extrude, revolve, holes, fillets) with a passing dimension table, not by gesturing at every possible part. Natural-language edits to the finished model are a stretch goal, gated behind a working core.
